# 종합소득세 신고 도우미 (jongsose-helper)

> **공식 종합소득세 책자(2025년 귀속)를 근거로 신고 절차·용어·서식을 즉답하는 RAG 챗봇.**
> 일반 LLM 의 어림짐작이 아니라 **책자 원문을 검색해 근거로 답해** 경비율·세율 같은 숫자가 안 틀립니다.

🔗 **라이브**: https://jongsose-helper.vercel.app
🛠 **스택**: Next.js (App Router) · OpenAI(gpt-4o / 임베딩) · RAG · Vision

---

## 문제 정의
종소세 신고 보조 현장에선 "추계가 뭐냐", "단순경비율 신고 어떻게 하냐" 같은 질문이 끊임없이 들어오는데, 매번 책자를 뒤지면 느리고 사람마다 답이 흔들립니다.
이 챗봇은 **공식 책자를 지식 근거로 즉답**해 일관성과 속도를 동시에 잡고, 홈택스 서식 캡처는 **Vision 으로 직접 읽어** 누락/오류를 검토합니다.

## 한 줄 아키텍처
```
질문(+서식 이미지)
   │
   ▼
[동의어 매핑] 일상어("월급쟁이","장부없이") → 도메인어("근로소득","추계")
   │
   ▼
[KB 검색] 책자 임베딩(kb.json) 코사인 검색 ──▶ 근거 절(page PNG + OCR)
   │                                              │
   ▼                                              │
[GPT 답변] 근거만 인용, 없으면 "책자에 없음" ◀────┘  (+이미지면 Vision 서식검토)
   │
   ▼
[thread 자동제목(LLM 6~14자 요약)] · 인용 카드 → /kb 뷰어 · 👍/👎 피드백
```

## 🎯 핵심 기술 결정 (면접 talking points)

1. **환각 방지가 최우선** — 책자에 근거가 없으면 그럴듯하게 지어내지 않고 **"책자에 해당 내용 없음"** 이라고 답하도록 강제. 세금 숫자(경비율·세율)는 틀리면 안 되는 도메인이라 RAG 근거를 truth source 로.
2. **일상어 → 도메인어 동의어 매핑** — 사용자가 "프리랜서/장부 없이"라고 쳐도 "사업소득/추계"로 매핑해 검색. 현장 표현과 세법 용어의 간극을 메움.
3. **멀티모달(Vision) 서식 검토** — 홈택스 화면 캡처를 `gpt-4o` 로 읽어 누락 항목/오류를 책자 근거와 대조. 이미지 첨부 시에도 RAG 가 같이 돌아 근거 카드 동시 노출.
4. **근거 추적 UX** — 답변의 인용 카드를 클릭하면 `/kb/[chunkId]` 뷰어로 이동해 **원문 페이지 PNG + OCR 텍스트**를 직접 확인. "AI 가 어디 보고 답했는지"를 사용자가 검증 가능.
5. **하루 MVP 스코프 관리** — "토요일 알바 전 하루 안에 굴러가는 봇"을 목표로 P0 에 집중. 완벽보다 현장 작동 우선의 의사결정.

## 🔐 권한 설계
관리자 승인 전 ID 는 **1회 체험 후 잠금**(Gate/Locked 컴포넌트). 무분별한 LLM 비용 소모 방지 + 내부 도구 성격에 맞춘 접근 통제.
