# 장부닥터 (Jangbu Doctor) — 세무조사 위험거래 사전 점검

> **통장 CSV 한 번 올리면 세무조사 위험 거래만 골라주는 AI 사전 점검 도구.**
> 세무사 자문을 대체하는 게 아니라, 1년 뒤 세무조사에서 터질 거래를 **매월 미리** 잡아내는 체크리스트 도구입니다.

🔗 **라이브**: https://jangbu-doctor.vercel.app
🛠 **스택**: Next.js · 룰 엔진(TS) · 법령 RAG · OpenAI · 통장 마스킹

---

## 문제 정의
세무사사무실에 기장을 위탁한 회사의 경리/경영지원팀은 **무엇이 위험한지 모른 채** 1년을 보내고, 세무조사에서야 접대비·차량비·가공거래 문제가 드러납니다.
장부닥터는 통장 CSV 를 올리면 **결정론적 룰 엔진**으로 위험 거래를 등급화하고(법령 근거 + 조치 제안), 필요 시 LLM 으로 보강합니다. "사후 발견 → 사전 점검"으로 포지셔닝.

## 한 줄 아키텍처
```
통장 CSV
   │
   ▼
[CSV 파서] 은행별 포맷 정규화 ─▶ [마스킹] 계좌·개인정보
   │
   ▼
[룰 엔진] 6개 카테고리 룰 병렬 평가
   ├ 접대비(골프/유흥)  ├ 차량비  ├ 개인경비  ├ 부가세  ├ 이상패턴  ├ 일반지출
   │   각 룰: detect() → level(high/medium/low/safe) + 법령근거 + 조치제안
   ▼
[위험 등급 집계] 거래별 overallLevel ─▶ [법령 RAG] 근거 절 매칭
   │
   ▼
[대시보드] 위험거래 리스트 · 처방전(prescription) · 엑셀 내보내기
```

## 🎯 핵심 기술 결정 (면접 talking points)

1. **세무 도메인 지식을 룰로 코드화** — 접대비(법인세법 §25), 차량비, 가공거래 패턴, 부가세 매입세액 불공제 등 **세무조사에서 실제 문제되는 항목**을 `Rule` 객체(`detect`/`explain`/`legalReference`/`getSuggestion`)로 모듈화. 세무·회계 5년 경력의 도메인 지식을 그대로 엔진에 녹임.
2. **결정론적 룰을 baseline, LLM 은 보강** — 위험 판단의 1차는 재현 가능한 룰 엔진. 각 거래에 걸린 룰 ID·법령근거·조치제안을 명시해 **"왜 위험한지"를 설명**. (susemi/netscope 와 공유하는 설계 철학)
3. **통장 데이터 마스킹** — 계좌번호·개인정보를 마스킹해 민감정보 노출 최소화. 금융 데이터를 다루는 도구의 기본기.
4. **법령 RAG 로 근거 강화** — 룰이 가리키는 법 조항을 RAG 로 끌어와 사용자가 근거를 확인하게.
5. **명확한 포지셔닝** — "세무사 대체 ❌ / 사전 체크리스트 ✅". 책임 범위를 분명히 하고 Disclaimer 로 오해 방지 — 규제 도메인 제품의 신중함.
